Stage 06 · Concurrent Tool Calls

HashMap tracks in-flight futures.
Results route back by task ID.

When agents fire multiple tool calls concurrently, a hash map of task_id → Future lets each result find its way home in O(1) — regardless of arrival order.

SCROLL
Overview
Concurrency without chaos

Without a tracking map, out-of-order async results can't be matched to their caller. The map is the glue that keeps concurrent execution deterministic.

🗺️
task_id → Future
Each spawned coroutine gets a unique ID. The hashmap stores ID → Future object. When a result arrives (in any order), O(1) lookup routes it to the right caller.
⚡
Parallel speedup
3 tool calls sequential = 3× latency. 3 concurrent = 1× latency of the slowest. asyncio.gather(), Promise.all(), Ray remote all use this pattern.
🔧
Cleanup on completion
On result or error, remove the future from the map. Track timeouts — hung tasks accumulate if never cleaned. O(1) delete from hashmap.

Python · asyncio futures
import asyncio, uuid

# HashMap: task_id → Future
pending: dict[str, asyncio.Future] = {}

async def call_tool(name, args):
    task_id = str(uuid.uuid4())[:8]
    future = asyncio.get_event_loop().create_future()
    pending[task_id] = future        # O(1) store

    # Fire and forget — result comes later
    asyncio.create_task(_execute(task_id, name, args))
    return task_id

async def on_result(task_id, result):
    future = pending.pop(task_id, None)  # O(1)
    if future:
        future.set_result(result)

# Fire 4 tools in parallel
ids = await asyncio.gather(
    call_tool("search_db", {...}),
    call_tool("fetch_api", {...}),
    call_tool("query_kb", {...}),
    call_tool("check_cache", {...}),
)

Key insights
What engineers get wrong
KEY INSIGHT — Timeouts and cancellation
Futures without timeouts = silent hangs. Always wrap async tool calls with asyncio.wait_for(coro, timeout=N). On TimeoutError, cancel the future and remove it from the map — otherwise the HashMap leaks references and the task stays "in-flight" forever. Budget timeouts per tool, not per request.
KEY INSIGHT — Out-of-order aggregation
asyncio.gather() collects all results but blocks until the slowest tool finishes. For streaming responses, use asyncio.as_completed() — it yields futures as they resolve. This lets you start building the response with fast results (cache hit at 0.8s) while slow tools (KB at 3s) are still running.
Error handling patterns
gather() with return_exceptions=True
Default: one tool exception cancels all others. With return_exceptions=True: exceptions become result values. Orchestrator checks each result type — isinstance(r, Exception) — and handles partial failures gracefully instead of all-or-nothing.
Cleanup on cancellation
When a Future is cancelled (timeout or upstream error), run cleanup: close HTTP connections, release DB cursors, decrement semaphore count. Use try/finally inside the async function — CancelledError propagates as an exception, so finally always runs.
Alternatives
Thread pool (concurrent.futures)
ThreadPoolExecutor + Future map. CPU-bound tasks or blocking I/O. GIL limits CPU parallelism but I/O releases it.
Use when: calling sync libraries (requests, psycopg2)
Message queue (Kafka/SQS)
Producer emits tool requests; workers consume and publish results. Natural fan-out, durable, retry-native. Results correlated by task_id in reply queue.
Use when: distributed workers, cross-service fan-out, durability needed
Reactor pattern (Node.js)
Single-threaded event loop with Promise.all(). JS AI SDKs (LangChain.js, Vercel AI SDK) use this natively. Same HashMap tracking pattern, different runtime.
Use when: JS/TS stack, edge runtime (Cloudflare Workers)
Real-world systems
OpenAI Assistants API
Tool calls returned as a batch in one run step. Client fires all in parallel, submits results via submit_tool_outputs. Internally uses Future map pattern.
Anthropic Tool Use
Model returns multiple tool_use blocks in one message. Client should fire all in parallel. Results submitted as tool_result blocks in the next turn.
LangChain LCEL
RunnableParallel executes branches concurrently via asyncio.gather. ToolNode automatically fans out multiple tool calls in one step.
Temporal.io
Durable execution framework. Parallel activity executions via Promise.all / workflow.all. State survives process crashes — future map persisted in workflow history.
