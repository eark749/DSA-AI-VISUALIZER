Overview
Two structures at the output stage

Ring buffer handles the mechanical streaming. Parse tree handles structural validation. Together they let you emit tokens to the user while guaranteeing the final output is well-formed.

🔄
Ring buffer
Fixed-size circular array. Write head advances on each new token. Read head trails it. O(1) write and read. No reallocation. Used for chunked HTTP streaming and SSE event buffering.
🌳
Incremental AST
JSON parser builds a tree as tokens arrive. Each '{', '[', '"' opens a node. Closing token completes it. Schema validation runs against partial tree — catches errors early.
✅
Early validation
If the model is outputting malformed JSON, the incremental parser detects it within the first few tokens — long before the full response arrives. Trigger retry immediately.

Python · Streaming + incremental parsing
from collections import deque
import ijson  # streaming JSON parser

class StreamBuffer:
    def __init__(self, size=8):
        self.buf = [None] * size        # ring array
        self.write = self.read = 0
        self.size = size

    def push(self, token):             # O(1)
        self.buf[self.write % self.size] = token
        self.write += 1

    def pop(self):                     # O(1)
        token = self.buf[self.read % self.size]
        self.read += 1
        return token

# Incremental JSON parse as tokens stream
buf = StreamBuffer()
