Overview
Two structures, one pipeline

They solve different halves of the routing problem — detection and dispatch — and compose naturally into a fast, transparent pipeline.

🌲
Step 1 — Trie scans
The prefix trie walks your raw text character by character to detect which intent keyword appears. O(L) where L is the keyword length.
⚡
Step 2 — Hash map dispatches
Once the intent string is extracted, a single O(1) dictionary lookup returns the correct agent class. Instant, no matter how many agents exist.
🤖
Step 3 — Agent handles
The resolved agent receives the original user query and processes it. The routing pipeline is complete in microseconds.

# Intent → Agent mapping (O(1) lookup)
router = {
    "billing":  BillingAgent,
    "tech":     SupportAgent,
    "refund":   RefundAgent,
    "account":  AccountAgent,
}

def route(intent: str):
    agent = router.get(intent, FallbackAgent)
    return agent.handle(user_input)

Data structure #2
Prefix trie — the detector

The trie scans raw text character by character. When it hits a terminal node, it has found the intent keyword and hands the string to the hash map.

Python · Trie detection
class TrieNode:
    def __init__(self):
        self.children = {}
        self.intent = None

class IntentTrie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, keyword, intent):
        node = self.root
        for char in keyword:
            node = node.children.setdefault(
                char, TrieNode())
        node.intent = intent

    def scan(self, text):
        for start in range(len(text)):
            node = self.root
            for ch in text[start:]:
                if ch not in node.children:
                    break
                node = node.children[ch]
                if node.intent:
                    return node.intent
        return "default"

# Full pipeline
trie = IntentTrie()
trie.insert("billing", "billing")
trie.insert("tech",    "tech")
trie.insert("refund",  "refund")

intent = trie.scan(user_input)
agent  = router[intent]
agent.handle(user_input)
Comparison
Which router to use?

The hash map is always present. The question is what feeds it — a trie scan, a regex, or an LLM.

Hash map only
for keyword in keys → O(1) dispatch
  • ✓Trivial to implement
  • ✓Fast enough for <20 intents
  • ✗O(k) keyword scan is sequential
  • ✗No prefix sharing
Trie + Hash map
O(L) scan → O(1) dispatch
  • ✓Efficient for large keyword sets
  • ✓Shared prefix paths
  • ✓Early exit on mismatch
  • ✗More code to maintain
LLM + Hash map
O(tokens) classify → O(1) dispatch
  • ✓Handles paraphrases
  • ✓No keyword maintenance
  • ✗100× slower than trie
  • ✗Costly at scale
