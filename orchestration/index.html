Agents as nodes. Handoffs as edges.
DAG controls the flow.

Directed acyclic graphs define which agents run, in what order, and which can run in parallel. No DAG = no reliable multi-agent coordination.

Overview
Why a DAG?

A directed acyclic graph guarantees: no cycles (agents don't loop forever), dependency tracking (B runs only after A), and parallelism (A and B run together if independent).

🔀
Topological ordering
BFS/DFS on the DAG gives execution order. Agents with no in-edges run first. Kahn's algorithm detects cycles — if a cycle exists, it's a deadlock.
⚡
Parallel branches
Independent branches (no shared dependency) execute concurrently. Fan-out from one node = spawn parallel agents. Fan-in = wait for all to complete.
🔍
Failure isolation
If one branch fails, only its downstream nodes are affected. The DAG structure makes it clear exactly which agents to retry or skip on error.

Python · DAG execution
from collections import deque

class DAG:
    def __init__(self):
        self.nodes = {}   # name → agent
        self.edges = {}   # name → [children]
        self.in_degree = {}

    def add_edge(self, u, v):
        self.edges[u].append(v)
        self.in_degree[v] += 1

    def topological_run(self):
        # Kahn's algorithm
        queue = deque([
            n for n, d in
            self.in_degree.items() if d == 0
        ])
        while queue:
            # Run all zero-degree nodes
            # in parallel (asyncio.gather)
            ready = list(queue); queue.clear()
            await asyncio.gather(*[
                self.nodes[n].run()
                for n in ready
            ])
            for n in ready:
                for child in self.edges[n]:
                    self.in_degree[child] -= 1
                    if self.in_degree[child]==0:
                        queue.append(child)
