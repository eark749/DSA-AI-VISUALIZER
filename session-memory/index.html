Stage 02 · LLM Context Management

Deque slides. LRU cache evicts.
Together they manage agent memory.

LLMs are stateless. Agents keep conversation alive with a sliding deque for short-term turns and an LRU cache for multi-session state.

See live demo Explore LRU cache

Overview
Two layers of memory

Short-term uses a deque with a token budget. Long-term uses an LRU cache mapping session IDs to state — hot sessions stay in RAM, cold ones evict.

📜
Deque — sliding window
append() new messages right, popleft() oldest when token limit exceeded. O(1) both ends. OpenAI Agents SDK TrimmingSession uses exactly this.
🗂️
LRU Cache — session store
HashMap + doubly-linked list. O(1) get and put. Most-recently-used sessions stay in memory. Least-recently-used evict when capacity fills.
⚡
Why both?
Deque manages single conversation context. LRU manages which of thousands of concurrent sessions stay warm in memory vs disk/Redis.

Python · Deque sliding window
from collections import deque

class TrimmingSession:
    def __init__(self, max_tokens=4096):
        self._items: deque = deque()
        self.max_tokens = max_tokens

    def add_message(self, role, content):
        self._items.append({
            "role": role,
            "content": content
        })
        # Evict oldest until within budget
        while self._count_tokens() > self.max_tokens:
            self._items.popleft()   # O(1)

    def get_context(self):
        return list(self._items)

    def _count_tokens(self):
        return sum(
            len(m["content"].split())
            for m in self._items
        )

# Usage — exactly OpenAI Agents SDK pattern
session = TrimmingSession(max_tokens=200)
session.add_message("user", "help with order")
session.add_message("assistant", "sure, order ID?")
context = session.get_context()
# → send context to LLM API call

Python · LRU Cache
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity=4):
        self.cap = capacity
        self.cache = OrderedDict()

    def get(self, session_id):
        if session_id not in self.cache:
            return None
        # Move to end = most recently used
        self.cache.move_to_end(session_id)
        return self.cache[session_id]

    def put(self, session_id, state):
        if session_id in self.cache:
            self.cache.move_to_end(session_id)
        self.cache[session_id] = state
        if len(self.cache) > self.cap:
            # popitem(last=False) = evict LRU
            self.cache.popitem(last=False)

# In practice: Redis with maxmemory-policy
# allkeys-lru does exactly this at scale
cache = LRUCache(capacity=1000)
cache.put("sess_abc", {"history": [...]})
state = cache.get("sess_abc")  # O(1)

Key insights
What engineers get wrong
KEY INSIGHT — Deque
The deque doesn't summarize — it forgets abruptly. Message at position N-1 stays; message at position N is gone entirely. If a user's name was in message 1 and the window slides past it, the LLM forgets their name mid-conversation. Always persist critical facts (user ID, preferences) separately.
KEY INSIGHT — LRU Cache
LRU eviction is access-pattern dependent. A user who returns after 30 min finds their session evicted; one who spams requests stays warm. Set TTL separately from LRU — Redis supports both. Use maxmemory-policy allkeys-lru for automatic eviction at scale.
Alternatives
Beyond the deque
Summarization
LLM summarizes old turns into a compressed paragraph before evicting. Keeps gist, loses detail. O(tokens) cost per summarization call. Used in Claude's long-context mode.
Vector memory
Embed each turn, store in HNSW index. Retrieve semantically relevant past turns on each query. Never truly forgets — retrieves. O(log N) per retrieval. MemGPT and mem0 use this pattern.
Redis Cluster at scale
When a single LRU cache can't hold all active sessions, shard across Redis Cluster nodes. Consistent hashing routes session_id to the right node. Handles millions of concurrent sessions.
Real-world systems
OpenAI Agents SDK
TrimmingSession uses collections.deque exactly as shown — popleft on token overflow.
Redis
maxmemory-policy allkeys-lru — entire keyspace as LRU cache. O(1) get/set with automatic eviction.
LangChain
ConversationBufferWindowMemory wraps a deque with configurable k window. ConversationSummaryMemory adds LLM summarization.
MemGPT / mem0
Replaces deque with vector store. Retrieves relevant past context instead of sliding window. No abrupt forgetting.
