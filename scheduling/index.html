Stage 05 · Tool Execution

Priority heap orders.
Queue ensures user tasks never wait.

Agent tool calls are tasks with priorities. A min-heap guarantees the highest-priority task always executes first — in O(log N) per operation.

See the heap Compare strategies
SCROLL
Overview
Why priority matters in AI agents

Without priorities, a background summarization job can block a user waiting for an API response. A heap fixes this in O(log N).

🏔️
Min-heap property
Parent priority ≤ children priorities. Root is always the highest-priority task. Insert and extract-min both O(log N) via heapify-up/down.
⚡
Critical path first
User-facing tool calls get priority 1. Background tasks get priority 5+. The heap automatically serves critical path first — no starvation of interactive tasks.
📊
Real implementation
Temporal workflow engine supports task priorities natively. Celery uses multiple queues (high/default/low). RabbitMQ has priority queue plugins — same heap concept.

Task queue (heap order)
Python · heapq
import heapq

# (priority, task_id, payload)
task_queue = []

def push_task(priority, task):
    heapq.heappush(task_queue,
        (priority, task["id"], task)
    )  # O(log N)

def pop_task():
    pri, _, task = heapq.heappop(task_queue)
    return task  # O(log N), always min
